# Requirements Archive: Auto Minuting v2.0

**Defined:** 2026-03-25
**Archived:** 2026-03-26
**Status:** ✅ All requirements addressed

## v2.0 Requirements

### 파일 관리 (File Management)

- [x] **FILE-01**: 사용자가 회의 레코드를 삭제하면 DB 레코드와 연관 파일(오디오, 전사, 회의록)이 함께 정리된다 — Phase 8 완료

### 삼성 녹음기 연동 (Samsung Recorder)

- [x] **SREC-01**: 사용자가 삼성 녹음앱에서 전사 텍스트를 공유하면 Auto Minuting이 이를 수신하여 회의록 생성 파이프라인에 진입시킨다 — Phase 9 완료
- [x] **SREC-02**: 삼성 녹음앱 전사 완료 시 자동 감지 가능성을 실기기에서 검증한다 — Phase 11 완료 (Partial Go: 오디오 감지 가능, 전사 직접 감지 불가)

### NotebookLM 연동

- [x] **NLMK-01**: 사용자가 회의록을 NotebookLM 앱에 공유 Intent로 전달할 수 있다 — Phase 10 완료
- [x] **NLMK-02**: 사용자가 앱 내에서 Custom Tabs로 NotebookLM 웹을 열 수 있다 — Phase 10 완료
- [x] **NLMK-03**: NotebookLM MCP 서버 API를 통한 노트북 생성/소스 추가 기능의 앱 내 구현 가능성을 검토한다 — Phase 10 완료 (MCP-REVIEW.md)

### Gemini 인증 (Auth)

- [x] **AUTH-01**: 사용자가 설정 화면에서 Gemini API 키를 입력/변경할 수 있고, 암호화되어 저장된다 — Phase 8 완료
- [x] **AUTH-02**: 사용자가 Google OAuth로 로그인하여 API 키 없이 Gemini를 사용할 수 있다 (기술 검증 포함) — Phase 12 완료 (실기기 E2E 대기)

### Plaud BLE

- [x] **PLUD-01**: Plaud 녹음기와 BLE 연결을 실기기에서 디버깅하여 실제 오디오 파일 수신이 동작하도록 한다 — Phase 13 완료 (SDK AAR 배치 + 실기기 대기)
