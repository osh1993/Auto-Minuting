## Overview

Share Intent로 여러 오디오 파일을 받았을 때 자동으로 하나의 파일로 합쳐 기존 STT → 회의록 파이프라인으로 처리하는 기능을 추가했다. 삼성 녹음앱의 M4A 파일을 여러 개 선택해 공유하면 MediaMuxer 기반으로 재인코딩 없이 합쳐지며, 첫 번째 파일명이 회의 제목으로 사용된다. Groq 25MB 제한 초과 시 엔진 변경을 안내하는 알림도 추가됐다.

## Milestone Summary

### Key Decisions

- [Phase 50-01]: WavMerger를 별도 object로 분리 — 독립적 단위 테스트 가능, ShareReceiverActivity와 관심사 분리
- [Phase 50-01]: 런타임에 Samsung Voice Recorder가 M4A(audio/mp4)를 출력함을 발견 → WavMerger(WAV 전용)에서 AudioMerger(MediaMuxer 기반, M4A 지원)로 교체
- [Phase 50-01]: Content URI를 cacheDir 임시 파일로 복사 후 MediaExtractor에 전달 — MediaExtractor는 절대 경로 필요, Content URI 직접 접근 불가
- [Phase 50-01]: handleMultipleAudioShare에서 processSharedAudio를 직접 호출하지 않고 별도 구현 — Content URI 수명 보장 + 이미 로컬 파일이므로 URI 재변환 불필요
- [Groq 크기 초과]: TranscriptionTriggerWorker에서 Groq 25MB 초과 에러 감지 → PipelineNotificationHelper.notifyFileTooLarge() 호출로 엔진 변경 안내

### Issues Resolved

- MERGE-01: 여러 M4A 파일 공유 → 자동 합치기 → 파이프라인 진입 동작 확인 (5개 파일 → 38MB M4A)
- MERGE-02: 합쳐진 파일명 = 첫 번째 파일명 (getDisplayName() 기반)
- MERGE-03: 합쳐진 단일 파일이 기존 TranscriptionTriggerWorker 파이프라인 통과
- Groq 25MB 초과 시 "전사 엔진 변경 필요" 알림 표시 (알림 권한 활성화 필요)
