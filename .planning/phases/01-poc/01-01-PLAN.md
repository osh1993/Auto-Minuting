   - adb로 기기에서 Plaud 앱 APK 추출: `adb shell pm path ai.plaud.note` 후 `adb pull`
   - jadx가 설치되어 있으면: `jadx -d output/ plaud.apk`

   - **파일 저장 경로**: `filesDir`, `getExternalFilesDir`, `MediaStore`, `Environment.getExternalStorageDirectory` 등의 사용 패턴 검색. Scoped Storage 여부 판단
   - **오디오 포맷**: WAV/MP3/기타 포맷 관련 코드 (MediaCodec, AudioFormat, 파일 확장자 패턴)
   - **BLE GATT 서비스/특성 UUID**: `UUID.fromString`, `BluetoothGattService`, `BluetoothGattCharacteristic` 검색
   - **BLE 통신 프로토콜**: 파일 전송 관련 GATT write/read/notify 패턴
   - **클라우드 API 엔드포인트**: `api.plaud.ai`, Retrofit 인터페이스 정의
4. Scoped Storage 분석 결과에 따라 FileObserver 방식의 가능 여부를 판단:
   - 내부 저장소(filesDir) 사용 -> FileObserver 불가, "FileObserver: NOT_FEASIBLE" 기록
   - 공유 저장소(MediaStore/Downloads) 사용 -> FileObserver 가능, "FileObserver: FEASIBLE" 기록
   - 앱별 외부 저장소(getExternalFilesDir) 사용 -> FileObserver 제한적, "FileObserver: LIMITED" 기록
5. apk-analysis.md에 다음 섹션 포함:

   - ## 오디오 포맷 (확인된 포맷, 인코딩)
   - ## BLE 프로토콜 (서비스 UUID, 특성 UUID, 통신 패턴)
   - ## 클라우드 API (엔드포인트, 인증 방식)
   - ## 결론 (FileObserver 가능 여부 판단)
