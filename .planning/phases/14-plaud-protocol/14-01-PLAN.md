    1. GitHub Releases에서 plaud-sdk AAR 다운로드 시도:
       - `gh release list -R Plaud-AI/plaud-sdk` 로 릴리즈 확인
       - `gh release download v0.2.6 -R Plaud-AI/plaud-sdk -D ./captures/` 로 AAR 다운로드

    2. SDK AAR을 jadx CLI로 디컴파일:
       - `jadx -d ./decompiled-sdk/ captures/plaud-sdk.aar` (jadx가 PATH에 없으면 설치 경로 확인)
       - jadx 미설치 시: GitHub Releases에서 jadx zip 다운로드, 압축 해제 후 사용

    3. 디컴파일된 코드에서 BLE 프로토콜 상수 검색:
       - UUID 패턴: `grep -ri "UUID\|uuid\|0000.*-0000-1000-8000" ./decompiled-sdk/`
       - GATT 관련: `grep -ri "BluetoothGatt\|GattService\|GattCharacteristic" ./decompiled-sdk/`
       - 파일 전송: `grep -ri "writeCharacteristic\|readCharacteristic\|setNotification\|WRITE\|READ\|NOTIFY" ./decompiled-sdk/`
       - 인증: `grep -ri "appKey\|appSecret\|token\|auth\|initSdk" ./decompiled-sdk/`
       - 파일 목록: `grep -ri "getFileList\|fileList\|FILE_LIST" ./decompiled-sdk/`

    4. 기존 스텁 코드(TntAgent.kt, BleCore.kt)와 SDK 실제 코드의 시그니처를 비교하여 차이점 기록

    5. sdk-analysis.md 문서 작성:
       - GATT 서비스 UUID 목록 (표 형식)
       - GATT 특성 UUID 목록 (Properties: Read/Write/Notify 포함)
       - 파일 전송 명령어/응답 패턴
       - 인증 흐름 (appKey/appSecret 사용 방식)
       - SDK와 기존 스텁 코드 비교 결과
       - 난독화 수준 평가

    주의: SDK AAR은 일반적으로 APK보다 난독화 수준이 낮으므로, APK 분석보다 먼저 수행한다 (Pitfall 5 회피, per RESEARCH.md).
    SDK AAR 다운로드가 불가능한 경우, GitHub 소스 코드에서 직접 분석하고 그 사실을 문서에 기록한다.

       - `jadx -d ./decompiled-app/ plaud-app.apk`

       - BLE API: `grep -ri "BluetoothGatt\|connectGatt\|discoverServices" ./decompiled-app/`
       - 파일 전송: `grep -ri "writeCharacteristic\|setCharacteristicNotification" ./decompiled-app/`
       - Wi-Fi Direct: `grep -ri "WifiP2p\|WiFiDirect\|WIFI_P2P" ./decompiled-app/` (per D-05, 부차적)
       - 에러 메시지: `grep -ri "connect.*fail\|transfer.*error\|sync.*error" ./decompiled-app/`

    4. Task 1의 SDK 분석 결과와 UUID 교차 검증:
       - SDK에서 발견한 UUID가 APK에서도 동일하게 사용되는지 확인
       - APK에만 있는 추가 UUID/프로토콜이 있는지 확인

    5. apk-analysis.md 문서 작성:
       - 난독화 수준 평가
       - BLE 연결 흐름 (스캔 → 연결 → 서비스 발견 → 특성 읽기/쓰기)
       - 파일 전송 시퀀스 (명령어 → 데이터 → 완료 확인)
       - Wi-Fi Direct 전환 흐름 (발견 시, per D-05)
       - SDK UUID와의 교차 검증 결과
       - 인증/암호화 관련 발견 사항
       - 프로토콜 암호화 여부 초기 판단 (per D-07 No-Go 조건 관련)
