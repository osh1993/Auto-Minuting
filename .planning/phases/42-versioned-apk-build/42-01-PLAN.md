**1. defaultConfig 내 versionName과 versionCode 변경:**

```kotlin
versionCode = 6

```

**2. android 블록 내부(buildFeatures 아래)에 applicationVariants를 사용한 출력 파일명 변경 추가:**

AGP 9.x에서는 `setProperty("archivesBaseName", ...)` 대신 `applicationVariants.all` 접근법을 사용한다. `android` 블록 닫기 직전에 추가:

```kotlin
    // Release APK 파일명에 버전 번호 포함 (AutoMinuting-v6.0-release.apk)
    applicationVariants.all {
        val variant = this
        outputs.all {
            val output = this as com.android.build.gradle.internal.api.BaseVariantOutputImpl
            output.outputFileName = "AutoMinuting-v${variant.versionName}-${variant.buildType.name}.apk"
        }
    }
```

주의: `com.android.build.gradle.internal.api.BaseVariantOutputImpl`은 AGP internal API이지만 파일명 변경에 널리 사용되는 검증된 패턴이다. AGP 9.x에서 작동하지 않을 경우 대안으로 `base.archivesName.set(...)` 사용:

대안 (applicationVariants가 안 될 경우):
```kotlin
// android 블록 내부 최상위에
base.archivesName.set("AutoMinuting-v${defaultConfig.versionName}")
```

두 접근법 중 `base.archivesName.set()`이 더 안정적이므로 이것을 우선 시도한다. `android {}` 블록 바로 아래에 추가:

```kotlin
base.archivesName.set("AutoMinuting-v${android.defaultConfig.versionName}")
```

즉, 최종적으로:
- `defaultConfig` 내에서 `versionCode = 6`, `versionName = "6.0"` 설정
- `android {}` 블록 닫은 직후에 `base.archivesName.set("AutoMinuting-v${android.defaultConfig.versionName}")` 추가

이렇게 하면 Gradle이 APK 출력 시 `AutoMinuting-v6.0-release.apk`로 생성한다.
