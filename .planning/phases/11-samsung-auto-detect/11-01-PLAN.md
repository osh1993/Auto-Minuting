  <action>
    spike/ 패키지에 삼성 녹음앱 파일 감지 프로토타입을 구현한다. 이 코드는 스파이크 전용이며 임시 코드이다.

    **SamsungRecorderObserver.kt:**
    - ContentObserver를 상속하여 onChange(selfChange, uri) 구현
    - onChange에서 최근 60초 이내 추가된 파일을 MediaStore.Audio.Media에서 쿼리
    - 쿼리 projection: _ID, DISPLAY_NAME, RELATIVE_PATH, DATE_ADDED, MIME_TYPE, SIZE
    - API 29+에서 OWNER_PACKAGE_NAME도 쿼리 (nullable 처리)
    - RELATIVE_PATH가 "Recordings/Voice Recorder/"를 포함하는지로 삼성 녹음앱 파일 필터링
    - MediaStore.Files도 별도로 쿼리하여 text/plain 파일이 등록되는지 확인 (전사 텍스트 감지 시도)
    - 감지 결과를 SharedFlow로 emit하여 UI에서 수신 가능하게 함
    - 로그: Log.d("SpikeObserver", ...) 태그 사용
    - lastCheckedTimestamp를 유지하여 중복 감지 방지

    **SpikeService.kt:**
    - Foreground Service로 구현 (foregroundServiceType="specialUse")
    - onCreate에서 SamsungRecorderObserver 2개 등록:
      1) MediaStore.Audio.Media.EXTERNAL_CONTENT_URI (오디오 파일)
      2) MediaStore.Files.getContentUri(MediaStore.VOLUME_EXTERNAL) (모든 파일)
    - 알림 채널 "spike_channel" 생성, 간단한 ongoing 알림 표시 ("삼성 녹음앱 감시 중")
    - onDestroy에서 Observer 해제
    - Handler(Looper.getMainLooper())를 Observer에 전달
    - companion object에 감지 결과 SharedFlow 노출 (Activity에서 collect)

    **AndroidManifest.xml 수정:**
    - FOREGROUND_SERVICE_SPECIAL_USE 퍼미션 추가
    - SpikeService 등록 (foregroundServiceType="specialUse")
    - SpikeLogActivity 등록
    - READ_MEDIA_AUDIO 퍼미션 추가 (API 33+에서 MediaStore 쿼리용)
    - READ_EXTERNAL_STORAGE 퍼미션 추가 (API 32 이하 폴백, maxSdkVersion="32")

    AccessibilityService, MANAGE_EXTERNAL_STORAGE는 절대 사용하지 않는다 (REQUIREMENTS.md Out of Scope).
