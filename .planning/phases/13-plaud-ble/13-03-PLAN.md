  <how-to-verify>
    실기기에서 다음 시나리오를 순서대로 테스트한다:

    **Step 1: BLE 스캔 테스트**
    1. 앱을 실기기에 설치한다 (adb install 또는 Android Studio Run)
    2. Plaud 녹음기 전원을 켠다
    3. DashboardScreen에서 "녹음기 연결 시작" 버튼을 터치한다
    4. BLE 권한 요청 다이얼로그가 표시되면 "허용"한다
    5. BLE 상태가 SCANNING -> DEVICE_FOUND로 변경되는지 확인한다
    6. BLE 로그 섹션에 "기기 발견" 이벤트가 기록되는지 확인한다
    - 예상 결과: 녹음기 SN 번호가 로그에 표시됨

    **Step 2: BLE 연결 테스트**
    7. DEVICE_FOUND 후 자동으로 connectionBLE가 호출되어 CONNECTING -> CONNECTED 상태로 전환되는지 확인한다
    8. BLE 로그에 "연결 성공" 이벤트가 기록되는지 확인한다
    - 예상 결과: 상태가 CONNECTED로 표시됨

    **Step 3: 파일 다운로드 테스트**
    9. 연결 후 파일 목록이 조회되는지 Logcat에서 확인한다 (tag: PlaudBLE)
    10. 오디오 파일이 WAV 포맷으로 다운로드되는지 확인한다
    11. 다운로드 진행률이 표시되는지 확인한다
    - 예상 결과: 앱 내부 저장소에 .wav 파일이 저장됨

    **Step 4: E2E 파이프라인 테스트**
    12. 다운로드 완료 후 Notification이 "오디오 저장 완료"로 업데이트되는지 확인한다
    13. WorkManager가 전사 작업을 시작하는지 Logcat에서 확인한다
    14. 회의 목록에 새 회의가 추가되고 파이프라인 상태가 진행되는지 확인한다
    - 예상 결과: AUDIO_RECEIVED -> TRANSCRIBING -> GENERATING_MINUTES -> COMPLETED

    **문제 발생 시 확인할 것:**
    - SecurityException: BLE 권한이 거부된 경우 -> 설정에서 권한 허용
    - handshakeWaitSure 반복: 기기 모델/펌웨어 불일치 -> 기기 모델 확인 후 인증 방식 변경
    - scanFail: 블루투스가 꺼져있거나 위치 권한이 없는 경우 -> 시스템 설정 확인
    - exportAudio onError: sessionId가 잘못되었거나 포맷 미지원 -> Logcat에서 상세 에러 확인
  </how-to-verify>
