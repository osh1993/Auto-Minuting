    신규 파일 PlaudBleAgentListener.kt를 생성한다.

    1. BleAgentListener 인터페이스를 구현하는 클래스를 만든다.
    2. 내부 상태를 StateFlow로 관리한다:
       - `connectionState: StateFlow<BleConnectionState>` (IDLE, SCANNING, CONNECTING, CONNECTED, DISCONNECTED, ERROR)
       - `discoveredDevice: StateFlow<BleDevice?>` (스캔에서 발견된 기기)
       - `lastError: StateFlow<String?>` (마지막 에러 메시지)
    3. BleConnectionState를 sealed interface 또는 enum class로 정의한다:
       IDLE, SCANNING, DEVICE_FOUND, CONNECTING, CONNECTED, DISCONNECTED, ERROR(reason: String)
    4. 콜백 구현:
       - scanBleDeviceReceiver(device): discoveredDevice에 기기 설정, 상태를 DEVICE_FOUND로 변경, 로그 출력 "기기 발견: ${device.sn}, RSSI: ${device.rssi}"
       - btStatusChange(sn, status): BluetoothStatus에 따라 connectionState 업데이트, CONNECTED/DISCONNECTED 로그
       - bleConnectFail(sn, reason): 상태를 ERROR로 변경, lastError에 이유 기록
       - handshakeWaitSure(sn, param): 로그만 출력 "핸드셰이크 대기: $sn, param: $param"
       - scanFail(reason): 상태를 ERROR로 변경, lastError에 스캔 실패 이유 기록
    5. 모든 콜백에서 Log.d("PlaudBLE", ...) 로 상세 로깅한다.
    6. CompletableDeferred를 활용하여 suspendCancellableCoroutine 없이도 연결 완료를 대기할 수 있도록 한다:
       - `awaitDeviceFound(): BleDevice` (스캔 완료 대기)
       - `awaitConnected()` (연결 완료 대기)
       각 메서드는 내부 CompletableDeferred를 생성하고, 해당 콜백에서 complete 한다.
       타임아웃은 호출자(PlaudSdkManager)에서 withTimeout으로 처리한다.
    7. `reset()` 메서드: 모든 상태를 초기화하여 새 스캔/연결 사이클을 준비한다.

    주석은 한국어로 작성한다. import는 Plaud SDK 패키지(com.nicebuild.sdk 등)를 사용한다.
    AAR이 없을 때 컴파일 에러가 발생할 수 있으므로, 파일 상단에 주석으로 "plaud-sdk.aar 배치 필수" 를 명시한다.
