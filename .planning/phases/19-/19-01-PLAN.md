   - 생성자에 `UserPreferencesRepository`와 `Application`(WorkManager용) 의존성 추가. Application 대신 `@ApplicationContext context: Context`를 Hilt로 주입받는 패턴 사용.
   - `generateMinutes(meetingId: Long)` 함수 추가:
     a. viewModelScope.launch 내에서 실행
     b. meetingRepository.getMeetingById(meetingId)로 Meeting 조회 (Flow이므로 .first()로 일회성 조회)
     c. meeting.transcriptPath가 null이면 로그 남기고 return
     d. userPreferencesRepository.getMinutesFormatOnce()로 현재 형식 가져오기
     e. OneTimeWorkRequestBuilder<MinutesGenerationWorker>()로 WorkRequest 생성:
        - KEY_MEETING_ID = meetingId
        - KEY_TRANSCRIPT_PATH = meeting.transcriptPath
        - KEY_MINUTES_FORMAT = minutesFormat.name
     f. WorkManager.getInstance(context).enqueue(workRequest)
   - import 추가: `android.content.Context`, `dagger.hilt.android.qualifiers.ApplicationContext`, `androidx.work.*`, `com.autominuting.worker.MinutesGenerationWorker`, `com.autominuting.data.preferences.UserPreferencesRepository`, `kotlinx.coroutines.flow.first`

   기존 `*/*` intent-filter 바로 앞에 다음을 추가:
   ```xml
   <intent-filter>
       <action android:name="android.intent.action.SEND" />
       <category android:name="android.intent.category.DEFAULT" />
       <data android:mimeType="audio/*" />
   </intent-filter>
   ```
   이렇게 하면 Android 시스템의 공유 시트에서 Auto Minuting이 음성 파일 공유 대상으로 더 높은 우선순위로 표시된다.
