# Plan 39-01: GroqSttEngine 구현

## Context

Groq Whisper API를 사용하는 STT 엔진을 SttEngine 인터페이스 구현체로 추가한다.
GeminiSttEngine 패턴을 그대로 복제하되, Gemini의 Base64 인라인 방식 대신 OkHttp MultipartBody로 오디오 파일을 전송한다.
Groq Free 티어는 25MB 파일 제한이 있으므로, 파일 크기 초과 시 명확한 에러 메시지를 반환한다.

이 플랜은 엔진 클래스와 API 키 저장만 구현한다. SttEngineType enum 확장, TranscriptionRepositoryImpl 연결, 설정 UI는 39-03에서 일괄 처리한다.

## Files to Read First

- `app/src/main/java/com/autominuting/data/stt/SttEngine.kt` — 구현할 인터페이스
- `app/src/main/java/com/autominuting/data/stt/GeminiSttEngine.kt` — 참고 패턴 (OkHttpClient 생성, 재시도, Result 반환)
- `app/src/main/java/com/autominuting/data/security/SecureApiKeyRepository.kt` — API 키 저장 패턴

## Tasks

### Task 1: SecureApiKeyRepository에 Groq API 키 저장 메서드 추가

**파일:** `app/src/main/java/com/autominuting/data/security/SecureApiKeyRepository.kt`

기존 `getGeminiApiKey()`/`saveGeminiApiKey()` 패턴을 그대로 복제하여 Groq API 키 관련 메서드를 추가한다.

**구현:**

1. companion object에 상수 추가:
   ```kotlin
   private const val KEY_GROQ_API = "groq_api_key"
   ```

2. 메서드 3개 추가 (getGeminiApiKey/saveGeminiApiKey/clearGeminiApiKey 패턴 복사):
   ```kotlin
   /** 저장된 Groq API 키를 반환한다. 없거나 초기화 실패 시 null. */
   fun getGroqApiKey(): String? =
       encryptedPrefs?.getString(KEY_GROQ_API, null)

   /** Groq API 키를 암호화하여 저장한다. */
   fun saveGroqApiKey(apiKey: String) {
       encryptedPrefs?.edit()?.putString(KEY_GROQ_API, apiKey)?.apply()
           ?: Log.w(TAG, "Groq API 키 저장 실패: EncryptedSharedPreferences 사용 불가")
   }

   /** 저장된 Groq API 키를 삭제한다. */
   fun clearGroqApiKey() {
       encryptedPrefs?.edit()?.remove(KEY_GROQ_API)?.apply()
   }
   ```

### Task 2: GroqSttEngine 클래스 생성

**파일:** `app/src/main/java/com/autominuting/data/stt/GroqSttEngine.kt`

GeminiSttEngine 구조를 따르되 Groq REST API 호출로 교체한다.

**구현:**

1. 패키지: `com.autominuting.data.stt`

2. 클래스 선언 — GeminiSttEngine과 동일한 DI 패턴:
   ```kotlin
   @Singleton
   class GroqSttEngine @Inject constructor(
       private val secureApiKeyRepository: SecureApiKeyRepository
   ) : SttEngine {
   ```

3. companion object 상수:
   ```kotlin
   private const val TAG = "GroqSttEngine"
   private const val API_URL = "https://api.groq.com/openai/v1/audio/transcriptions"
   private const val MODEL = "whisper-large-v3-turbo"
   private const val MAX_FILE_SIZE = 25 * 1024 * 1024L  // 25MB (Free 티어 제한)
   private const val MAX_RETRIES = 3
   private const val RETRY_DELAY_MS = 20_000L
   ```

4. MIME 타입 맵 (GeminiSttEngine과 동일):
   ```kotlin
   private val MIME_MAP = mapOf(
       "m4a" to "audio/mp4",
       "mp4" to "audio/mp4",
       "mp3" to "audio/mpeg",
       "wav" to "audio/wav",
       "ogg" to "audio/ogg",
       "flac" to "audio/flac",
       "aac" to "audio/aac"
   )
   ```

5. `engineName()` 반환값: `"Groq Whisper (Cloud)"`

6. `isAvailable()`: `secureApiKeyRepository.getGroqApiKey()?.isNotBlank() == true`

7. `transcribe(audioFilePath: String, onProgress: (Float) -> Unit)` 오버라이드 구현 — 클라우드 엔진이므로 onProgress는 파라미터만 선언하고 호출하지 않는다. `withContext(Dispatchers.IO)` 안에서:
   - API 키 검증: 비어있으면 `Result.failure(GroqSttException("Groq API 키가 설정되지 않았습니다"))`
   - 파일 존재 확인
   - 파일 크기 검사: `MAX_FILE_SIZE` 초과 시 `Result.failure(GroqSttException("파일 크기(${fileSize/1024/1024}MB)가 Groq 제한(25MB)을 초과합니다. 압축된 형식(M4A/MP3)으로 변환하거나 Groq Dev 플랜으로 업그레이드하세요."))`
   - MIME 타입 결정: `MIME_MAP[extension]` 없으면 실패
   - OkHttpClient 생성 (GeminiSttEngine과 동일한 타임아웃):
     ```kotlin
     val client = OkHttpClient.Builder()
         .connectTimeout(60, TimeUnit.SECONDS)
         .writeTimeout(5, TimeUnit.MINUTES)
         .readTimeout(5, TimeUnit.MINUTES)
         .build()
     ```
   - MultipartBody 생성:
     ```kotlin
     val requestBody = MultipartBody.Builder()
         .setType(MultipartBody.FORM)
         .addFormDataPart("file", audioFile.name,
             audioFile.asRequestBody(mimeType.toMediaType()))
         .addFormDataPart("model", MODEL)
         .addFormDataPart("language", "ko")
         .addFormDataPart("response_format", "verbose_json")
         .build()
     ```
   - Request 생성:
     ```kotlin
     val request = Request.Builder()
         .url(API_URL)
         .header("Authorization", "Bearer $apiKey")
         .post(requestBody)
         .build()
     ```
   - 재시도 루프 (GeminiSttEngine 패턴 복사): HTTP 429 또는 SocketTimeoutException 시 `RETRY_DELAY_MS` 후 재시도
   - 응답 파싱: `JSONObject(responseBody).getString("text")`
   - 빈 응답 처리: text가 null/blank이면 실패
   - 성공 시 `Result.success(text.trim())`

8. 예외 클래스:
   ```kotlin
   class GroqSttException(
       message: String,
       cause: Throwable? = null
   ) : Exception(message, cause)
   ```
