# Plan 39-02: DeepgramSttEngine 구현

## Context

Deepgram Nova-3 API를 사용하는 STT 엔진을 SttEngine 인터페이스 구현체로 추가한다.
Groq과 달리 multipart가 아닌 binary body로 오디오를 직접 전송하며, 인증 헤더가 `Token` 프리픽스(Bearer가 아님)인 점에 주의한다.
응답 JSON의 텍스트 추출 경로가 깊다: `results.channels[0].alternatives[0].transcript`.

이 플랜은 엔진 클래스와 API 키 저장만 구현한다. enum/Repository 연결은 39-03에서 일괄 처리한다.

## Files to Read First

- `app/src/main/java/com/autominuting/data/stt/SttEngine.kt` — 구현할 인터페이스
- `app/src/main/java/com/autominuting/data/stt/GeminiSttEngine.kt` — 참고 패턴
- `app/src/main/java/com/autominuting/data/security/SecureApiKeyRepository.kt` — API 키 저장 패턴

## Tasks

### Task 1: SecureApiKeyRepository에 Deepgram API 키 저장 메서드 추가

**파일:** `app/src/main/java/com/autominuting/data/security/SecureApiKeyRepository.kt`

**구현:**

1. companion object에 상수 추가:
   ```kotlin
   private const val KEY_DEEPGRAM_API = "deepgram_api_key"
   ```

2. 메서드 3개 추가 (기존 패턴 복사):
   ```kotlin
   /** 저장된 Deepgram API 키를 반환한다. */
   fun getDeepgramApiKey(): String? =
       encryptedPrefs?.getString(KEY_DEEPGRAM_API, null)

   /** Deepgram API 키를 암호화하여 저장한다. */
   fun saveDeepgramApiKey(apiKey: String) {
       encryptedPrefs?.edit()?.putString(KEY_DEEPGRAM_API, apiKey)?.apply()
           ?: Log.w(TAG, "Deepgram API 키 저장 실패: EncryptedSharedPreferences 사용 불가")
   }

   /** 저장된 Deepgram API 키를 삭제한다. */
   fun clearDeepgramApiKey() {
       encryptedPrefs?.edit()?.remove(KEY_DEEPGRAM_API)?.apply()
   }
   ```

**주의:** 39-01에서 이미 Groq 관련 메서드를 추가했으므로, 그 아래에 이어서 추가한다. 39-01이 먼저 실행되지 않았다면 companion object에 KEY_GROQ_API도 함께 추가되어 있지 않을 수 있다 — 39-02는 Deepgram만 담당하므로 Groq 코드가 없어도 문제없다.

### Task 2: DeepgramSttEngine 클래스 생성

**파일:** `app/src/main/java/com/autominuting/data/stt/DeepgramSttEngine.kt`

**구현:**

1. 패키지: `com.autominuting.data.stt`

2. 클래스 선언:
   ```kotlin
   @Singleton
   class DeepgramSttEngine @Inject constructor(
       private val secureApiKeyRepository: SecureApiKeyRepository
   ) : SttEngine {
   ```

3. companion object 상수:
   ```kotlin
   private const val TAG = "DeepgramSttEngine"
   private const val BASE_URL = "https://api.deepgram.com/v1/listen"
   private const val MAX_RETRIES = 3
   private const val RETRY_DELAY_MS = 20_000L
   ```

4. MIME 타입 맵 (동일):
   ```kotlin
   private val MIME_MAP = mapOf(
       "m4a" to "audio/mp4",
       "mp4" to "audio/mp4",
       "mp3" to "audio/mpeg",
       "wav" to "audio/wav",
       "ogg" to "audio/ogg",
       "flac" to "audio/flac",
       "aac" to "audio/aac"
   )
   ```

5. `engineName()` 반환값: `"Deepgram Nova-3 (Cloud)"`

6. `isAvailable()`: `secureApiKeyRepository.getDeepgramApiKey()?.isNotBlank() == true`

7. `transcribe(audioFilePath: String, onProgress: (Float) -> Unit)` 오버라이드 구현 — 클라우드 엔진이므로 onProgress는 파라미터만 선언하고 호출하지 않는다. `withContext(Dispatchers.IO)` 안에서:
   - API 키 검증: 비어있으면 `Result.failure(DeepgramSttException("Deepgram API 키가 설정되지 않았습니다"))`
   - 파일 존재 확인
   - MIME 타입 결정
   - OkHttpClient 생성 (동일 타임아웃)
   - **핵심 차이점 — binary body 방식:**
     ```kotlin
     val requestBody = audioFile.asRequestBody(mimeType.toMediaType())

     val url = "$BASE_URL?model=nova-3&language=ko&smart_format=true&punctuate=true"
     val request = Request.Builder()
         .url(url)
         .header("Authorization", "Token $apiKey")  // Bearer가 아닌 Token!
         .post(requestBody)
         .build()
     ```
   - 재시도 루프 (GeminiSttEngine 패턴)
   - **응답 JSON 파싱 — 깊은 경로:**
     ```kotlin
     val json = JSONObject(responseBody)
     val text = json
         .getJSONObject("results")
         .getJSONArray("channels")
         .getJSONObject(0)
         .getJSONArray("alternatives")
         .getJSONObject(0)
         .getString("transcript")
         .trim()
     ```
   - 빈 응답 처리
   - `Result.success(text)`

8. 예외 클래스:
   ```kotlin
   class DeepgramSttException(
       message: String,
       cause: Throwable? = null
   ) : Exception(message, cause)
   ```
