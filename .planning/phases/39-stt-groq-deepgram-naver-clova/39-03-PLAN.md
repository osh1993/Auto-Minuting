# Plan 39-03: NaverClovaSttEngine + 통합 배선

## Context

Naver CLOVA Speech (Long) API를 사용하는 STT 엔진을 추가하고, 동시에 3개 신규 엔진(Groq, Deepgram, Naver CLOVA)을 기존 시스템에 통합한다.

CLOVA Speech는 다른 엔진과 달리:
1. invoke URL이 사용자별로 다르므로 API 키 외에 invoke URL도 저장해야 한다
2. 인증 헤더가 `X-CLOVASPEECH-API-KEY`이다
3. multipart/form-data이되, params를 JSON 문자열로 form field에 포함한다
4. sync 모드로 동작하여 긴 오디오 응답에 시간이 걸릴 수 있다 (readTimeout 10분)

통합 작업:
- SttEngineType enum에 GROQ, DEEPGRAM, NAVER_CLOVA 추가
- TranscriptionRepositoryImpl 생성자에 3개 엔진 주입 + when 분기 추가
- Hilt는 @Singleton @Inject constructor로 자동 등록 (SttModule 수정 불필요)

### Task 2: SttEngineType enum 확장 + TranscriptionRepositoryImpl 통합

**파일 1:** `app/src/main/java/com/autominuting/domain/model/SttEngineType.kt`

기존 enum에 3개 항목 추가:
```kotlin
enum class SttEngineType {
    /** Gemini 2.5 Flash 멀티모달 API (클라우드, 기본값) */
    GEMINI,
    /** Whisper (whisper.cpp 온디바이스) */
    WHISPER,
    /** Groq Whisper API (클라우드) */
    GROQ,
    /** Deepgram Nova-3 (클라우드) */
    DEEPGRAM,
    /** Naver CLOVA Speech (클라우드) */
    NAVER_CLOVA
}
```

**주의:** UserPreferencesRepository의 `sttEngineType` Flow에서 `SttEngineType.valueOf(name)`을 호출한다. 기존 사용자가 저장한 값(GEMINI, WHISPER)은 그대로 유효하므로 호환성 문제 없음. 만약 알 수 없는 값이 저장되어 있으면 IllegalArgumentException이 발생할 수 있으나, 기존 코드가 이미 `prefs[STT_ENGINE_KEY] ?: SttEngineType.GEMINI.name`으로 기본값 처리하고 있어 null인 경우만 방어됨. valueOf 실패를 방어하려면 UserPreferencesRepository의 sttEngineType Flow에 try-catch를 추가한다:

```kotlin
// UserPreferencesRepository.kt의 sttEngineType Flow 수정
val sttEngineType: Flow<SttEngineType> = dataStore.data.map { prefs ->
    val name = prefs[STT_ENGINE_KEY] ?: SttEngineType.GEMINI.name
    try {
        SttEngineType.valueOf(name)
    } catch (e: IllegalArgumentException) {
        SttEngineType.GEMINI  // 알 수 없는 값 → 기본값 폴백
    }
}
```

같은 패턴으로 `getSttEngineTypeOnce()`도 수정:
```kotlin
suspend fun getSttEngineTypeOnce(): SttEngineType =
    dataStore.data.first().let { prefs ->
        val name = prefs[STT_ENGINE_KEY] ?: SttEngineType.GEMINI.name
        try {
            SttEngineType.valueOf(name)
        } catch (e: IllegalArgumentException) {
            SttEngineType.GEMINI
        }
    }
```
