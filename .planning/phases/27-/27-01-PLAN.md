   ```kotlin
   }
   ```

2. **`fun downloadFromUrl(url: String)`** 함수 추가:
   - URL 유효성 검증: `url.isBlank()` 체크, `http://` 또는 `https://`로 시작하는지 확인. 실패 시 `DownloadState.Error("올바른 URL을 입력해주세요")` 설정 후 return
   - 이미 Downloading 상태이면 중복 실행 방지 (return)
   - `viewModelScope.launch(Dispatchers.IO)` 블록 내에서:
     a. `_downloadState.value = DownloadState.Downloading(0f)` 설정
     b. OkHttpClient (WhisperModelManager 패턴 재사용): connectTimeout 30초, readTimeout 60초
     c. `Request.Builder().url(url).build()` 로 GET 요청
     d. 응답 실패 시 `DownloadState.Error("다운로드 실패: HTTP ${response.code}")` 설정
     e. URL에서 파일명 추출: `url.substringAfterLast("/").substringBefore("?")`. 비어있으면 `"download_${System.currentTimeMillis()}"` 사용
     f. 확장자 추출: 파일명에서 `.substringAfterLast(".", "m4a")`. 지원 확장자(m4a, mp3, wav, ogg, flac, mp4) 아니면 Content-Type 헤더에서 추출, 기본값 "m4a"
     g. audio 디렉터리 (`File(context.filesDir, "audio")`) 에 임시 파일(`url_${now}.downloading`) 생성
     h. WhisperModelManager와 동일한 8KB 버퍼 + 진행률 패턴으로 다운로드
     i. 완료 후 `url_${now}.{extension}`으로 rename
     j. `_downloadState.value = DownloadState.Processing` 설정
     k. ShareReceiverActivity.processSharedAudio() 패턴 따라:
        - Meeting 생성: `title = 파일명(확장자 제거)`, `source = "URL_DOWNLOAD"`, `pipelineStatus = AUDIO_RECEIVED`, `audioFilePath = 최종파일경로`
        - `meetingRepository.insertMeeting(meeting)` 으로 DB 저장
        - `TranscriptionTriggerWorker` enqueue (KEY_AUDIO_FILE_PATH, KEY_MEETING_ID)
     l. `_downloadState.value = DownloadState.Idle` 설정 (성공 시)
   - catch 블록: `DownloadState.Error("다운로드 오류: ${e.message}")`, 임시 파일 삭제

3. **`fun clearDownloadError()`** 함수 추가: `_downloadState.value = DownloadState.Idle`

3. **URL 다운로드 카드** — NotebookLM 카드와 테스트 도구 카드 사이에 배치 (`Spacer(modifier = Modifier.height(16.dp))` 후):
   - `Card` (fillMaxWidth, padding horizontal 16.dp, containerColor = `MaterialTheme.colorScheme.secondaryContainer`)
   - Card 내부 Column(padding 16.dp):
     a. `Text("URL에서 음성 다운로드", style = titleMedium)`
     b. `Text("음성 파일 URL을 입력하면 다운로드 후 자동 전사합니다", style = bodySmall, color = onSecondaryContainer)`
     c. `Spacer(height = 8.dp)`
     d. `OutlinedTextField`:
        - value = urlText, onValueChange = { urlText = it }
        - label = { Text("음성 파일 URL") }
        - placeholder = { Text("https://example.com/audio.m4a") }
        - singleLine = true
        - modifier = Modifier.fillMaxWidth()
        - enabled = downloadState !is DownloadState.Downloading (다운로드 중 입력 비활성화)
     e. `Spacer(height = 8.dp)`
     f. 다운로드 상태에 따른 UI 분기:
        - `DownloadState.Idle` → Button("다운로드 시작", fillMaxWidth, enabled = urlText.isNotBlank(), onClick = { viewModel.downloadFromUrl(urlText) })
        - `DownloadState.Downloading(progress)` →
          - `LinearProgressIndicator(progress = { progress }, modifier = Modifier.fillMaxWidth())`
          - `Text("다운로드 중... ${(progress * 100).toInt()}%", style = bodySmall)`
        - `DownloadState.Processing` →
          - `LinearProgressIndicator(modifier = Modifier.fillMaxWidth())` (indeterminate)
          - `Text("전사 파이프라인 진입 중...", style = bodySmall)`
        - `DownloadState.Error(message)` →
          - `Text(message, style = bodySmall, color = MaterialTheme.colorScheme.error)`
          - `Spacer(height = 4.dp)`
          - `OutlinedButton("다시 시도", onClick = { viewModel.clearDownloadError() })` (에러 클리어 후 재시도 가능)
     g. 다운로드 성공(Idle 복귀) 시 urlText를 클리어하려면: `LaunchedEffect(downloadState)` 에서 이전이 Processing이고 현재가 Idle이면 urlText = "" (또는 간단하게 다운로드 시작 시 별도 flag로 관리)
        - 간단한 방법: downloadFromUrl 호출 직후 성공 완료 콜백 대신, DownloadState.Idle 전환 시 urlText 클리어는 하지 않아도 됨 (사용자가 같은 URL 다시 시도할 수 있으므로). 이 부분은 생략.
