`app/src/main/java/com/autominuting/data/minutes/DeepgramMinutesEngine.kt` 파일을 새로 생성한다.

**클래스 구조:**
- package: `com.autominuting.data.minutes`
- `@Singleton class DeepgramMinutesEngine @Inject constructor(private val secureApiKeyRepository: SecureApiKeyRepository) : MinutesEngine`
- companion object에 TAG, ENDPOINT(`https://api.deepgram.com/v1/read`), MAX_RETRIES(3), RETRY_DELAY_MS(20_000L) 상수 정의

**engineName():**
- 반환값: `"Deepgram Intelligence (영어만)"` — 한국어 미지원 안내 포함 (per MIN6-01)

**isAvailable():**
- `secureApiKeyRepository.getDeepgramApiKey()?.isNotBlank() == true` 반환

**generate(transcriptText, customPrompt):**
- `customPrompt` 파라미터는 무시한다. Deepgram API는 자체 요약 로직을 사용하므로 커스텀 프롬프트 적용 불가. 로그에 경고 기록: `Log.d(TAG, "Deepgram은 커스텀 프롬프트를 지원하지 않습니다 — 무시합니다")`
- API 키 조회: `secureApiKeyRepository.getDeepgramApiKey()` -> null이면 `Result.failure(IllegalStateException("Deepgram API 키가 설정되지 않았습니다"))` 반환

- `withContext(Dispatchers.IO)` 블록 안에서 실행

- OkHttpClient 생성: connectTimeout 30초, readTimeout 60초, writeTimeout 30초
- URL: `"$ENDPOINT?summarize=true&language=en"`
- Request 구성:
  - POST 메서드
  - Header: `"Authorization"` to `"Token $apiKey"` (Bearer가 아님 — Deepgram 고유 방식)
  - Header: `"Content-Type"` to `"application/json"`
  - Body: `JSONObject().put("text", transcriptText).toString()` — okhttp3.RequestBody.Companion.toRequestBody 사용, MediaType `"application/json".toMediaType()`

- 재시도 로직 (DeepgramSttEngine 패턴과 동일):
  ```
  for (attempt in 1..MAX_RETRIES) {
      // request 빌드 및 execute
      // HTTP 429 -> delay(RETRY_DELAY_MS) 후 continue
      // !response.isSuccessful -> Result.failure
      // 성공 -> JSON 파싱 -> break
      // SocketTimeoutException -> delay 후 재시도
  }
  ```

- 응답 JSON 파싱:
  - `val json = JSONObject(responseBody)`
  - `val results = json.getJSONObject("results")`
  - `val summary = results.getJSONObject("summary")`
  - 요약 텍스트: `summary.optString("short")` 먼저 시도, 비어있으면 `summary.optString("text")` 폴백 시도
  - 여전히 비어있으면 `Result.failure(IllegalStateException("Deepgram 요약 결과가 비어있습니다"))` 반환

- 성공 시: `Result.success(summaryText)` 반환
- 모든 재시도 소진 시: `Result.failure(...)` 반환
