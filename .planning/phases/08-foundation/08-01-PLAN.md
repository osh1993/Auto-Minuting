    **1. MeetingRepositoryImpl.kt — deleteMeeting() 파일 삭제 로직 추가** (per D-03):
    - 기존 `override suspend fun deleteMeeting(id: Long) = meetingDao.delete(id)` 를 다음으로 교체:
      ```kotlin
      override suspend fun deleteMeeting(id: Long) {
          // 1. 파일 경로 조회 (삭제 전에 Entity에서 가져와야 함)
          val entity = meetingDao.getMeetingByIdOnce(id) ?: return

          // 2. 연관 파일 삭제 (실패해도 DB 삭제 진행 — 고아 파일 > 고아 레코드)
          listOfNotNull(
              entity.audioFilePath,
              entity.transcriptPath,
              entity.minutesPath
          ).forEach { path ->
              try { File(path).delete() } catch (_: Exception) { }
          }

          // 3. DB 레코드 삭제
          meetingDao.delete(id)
      }
      ```
    - `audioFilePath`는 non-null이지만 빈 문자열일 수 있으므로 `listOfNotNull`로 처리. 실제로는 `listOf(entity.audioFilePath) + listOfNotNull(entity.transcriptPath, entity.minutesPath)` 가 더 정확하나, `listOfNotNull`은 null을 필터링하므로 non-null인 audioFilePath도 안전하게 포함됨.
