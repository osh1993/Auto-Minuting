  <action>
    1. `data/quota/GeminiQuotaTracker.kt` 생성:
       - `enum class QuotaCategory { STT, MINUTES }` — API 호출 종류 구분
       - `data class QuotaUsage(val sttCount: Int, val minutesCount: Int, val dailyLimit: Int, val date: String)` — 날짜별 사용량
         - `val totalCount: Int get() = sttCount + minutesCount`
         - `val usagePercent: Float get() = totalCount.toFloat() / dailyLimit`
         - `val isOverThreshold: Boolean get() = usagePercent >= 0.9f`
         - `val remaining: Int get() = (dailyLimit - totalCount).coerceAtLeast(0)`
       - `@Singleton class GeminiQuotaTracker @Inject constructor(dataStore: DataStore<Preferences>)`
       - companion object에 DataStore 키 정의:
         - `QUOTA_DATE_KEY = stringPreferencesKey("gemini_quota_date")` — 카운트 날짜 (yyyy-MM-dd)
         - `QUOTA_STT_COUNT_KEY = intPreferencesKey("gemini_quota_stt_count")`
         - `QUOTA_MINUTES_COUNT_KEY = intPreferencesKey("gemini_quota_minutes_count")`
         - `DAILY_LIMIT = 1500` — Gemini Free 일일 RPD 한도
       - `val usage: Flow<QuotaUsage>` — dataStore.data.map으로 현재 사용량을 QuotaUsage로 매핑. 날짜가 오늘과 다르면 카운트 0으로 반환 (자동 리셋 효과)
       - `suspend fun recordUsage(category: QuotaCategory)` — dataStore.edit로 카운트 증가. 날짜가 오늘과 다르면 먼저 리셋 후 1로 설정. 날짜 비교: `java.time.LocalDate.now().toString()`
       - 날짜 포맷은 `LocalDate.now().toString()` (ISO-8601 yyyy-MM-dd)

    2. `di/QuotaModule.kt` 생성:
       - `@Module @InstallIn(SingletonComponent::class) object QuotaModule`
       - GeminiQuotaTracker는 @Singleton + @Inject constructor이므로 별도 @Provides 불필요
       - 이 모듈은 향후 확장용으로 빈 모듈로 생성하되, 실제로 Hilt가 DataStore<Preferences>를 이미 DataStoreModule에서 제공하므로 GeminiQuotaTracker는 자동 주입됨
       - 참고: QuotaModule이 불필요하면 생략하고 GeminiQuotaTracker의 @Singleton @Inject만으로 충분

    3. `GeminiSttEngine.kt` 수정:
       - 생성자에 `private val quotaTracker: GeminiQuotaTracker` 추가
       - `transcribe()` 메서드에서 API 호출 성공 시 (Result.success 반환 직전) `quotaTracker.recordUsage(QuotaCategory.STT)` 호출
       - 429 재시도 시에도 실제 성공한 호출만 카운트 (재시도 자체는 카운트하지 않음)

    4. `MinutesRepositoryImpl.kt` 수정:
       - 생성자에 `private val quotaTracker: GeminiQuotaTracker` 추가
       - `generateMinutes()` 메서드에서 `result.isSuccess` 블록 안에서 `quotaTracker.recordUsage(QuotaCategory.MINUTES)` 호출
