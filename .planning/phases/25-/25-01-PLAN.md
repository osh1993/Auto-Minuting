```kotlin
/** 전사 항목의 제목을 변경한다. */
fun updateTitle(meetingId: Long, newTitle: String) {
    viewModelScope.launch {
        val meeting = meetingRepository.getMeetingById(meetingId).first()
        if (meeting != null) {
            meetingRepository.updateMeeting(
                meeting.copy(title = newTitle, updatedAt = Instant.now())
            )
        }
    }
}
```
- import `java.time.Instant` (이미 있을 수 있음 — 확인 후 추가)

   ```kotlin
   var meetingToRename by remember { mutableStateOf<Meeting?>(null) }
   ```

   b. TranscriptMeetingCard에 `onRenameRequest: (Meeting) -> Unit` 파라미터 추가.

      기존 카드 전체 클릭(onClick)과 구분되도록, 제목 Text에 별도 `.clickable { onRenameRequest(meeting) }` 적용.

   c. TranscriptsScreen에서 TranscriptMeetingCard 호출 시:
   ```kotlin
   onRenameRequest = { m -> meetingToRename = m }
   ```

   d. 이름 편집 AlertDialog 추가 (meetingToDelete 다이얼로그 패턴 따름):
   ```kotlin
   meetingToRename?.let { meeting ->
       var editedName by remember(meeting.id) { mutableStateOf(meeting.title) }
       AlertDialog(
           onDismissRequest = { meetingToRename = null },
           title = { Text("이름 편집") },
           text = {
               OutlinedTextField(
                   value = editedName,
                   onValueChange = { editedName = it },
                   label = { Text("전사 이름") },
                   singleLine = true,
                   modifier = Modifier.fillMaxWidth()
               )
           },
           confirmButton = {
               TextButton(
                   onClick = {
                       val trimmed = editedName.trim()
                       if (trimmed.isNotBlank()) {
                           viewModel.updateTitle(meeting.id, trimmed)
                       }
                       meetingToRename = null
                   }
               ) { Text("확인") }
           },
           dismissButton = {
               TextButton(onClick = { meetingToRename = null }) { Text("취소") }
           }
       )
   }
   ```

   e. 필요한 import 추가:

제목 Text의 clickable에서 기존 카드 onClick 전파를 방지하기 위해 제목 Text 안쪽에 직접 `.clickable` 적용하되, 카드 전체 clickable은 유지한다. Compose에서 내부 clickable은 외부로 이벤트를 전파하지 않으므로 별도 stopPropagation 불필요.
