**UnauthorizedException** 정의: `class UnauthorizedException(message: String) : Exception(message)` — 같은 파일에 선언.

**DriveUploadRepository** 구현:
- `@Singleton` + `@Inject constructor(private val okHttpClient: OkHttpClient)` — OkHttpClient는 기존 NetworkModule에서 이미 제공됨.
- `private val UPLOAD_URL = "https://www.googleapis.com/upload/drive/v3/files?uploadType=multipart"`
- `suspend fun uploadFile(accessToken, fileName, mimeType, fileContent, parentFolderId): Result<String>` — `withContext(Dispatchers.IO)` 내에서 실행.

**multipart/related 바디 구성 (`buildMultipartBody` private 메서드):**
```
boundary = "auto_minuting_boundary_${System.currentTimeMillis()}"
메타데이터 파트: Content-Type: application/json; charset=UTF-8
  {"name":"$fileName","parents":["$parentFolderId"]}
파일 파트: Content-Type: $mimeType
  fileContent 바이트 배열
구분자: --$boundary (각 파트 앞), --$boundary-- (마지막)
줄바꿈: \r\n
```
구체적 구현: ByteArray 리스트를 `fold(ByteArray(0)) { acc, bytes -> acc + bytes }` 로 합산.

**Request 구성:**
- `addHeader("Authorization", "Bearer $accessToken")`
- `addHeader("Content-Type", "multipart/related; boundary=$boundary")`
- `.post(body.toRequestBody("multipart/related; boundary=$boundary".toMediaTypeOrNull()))`

**응답 처리:**
- `response.isSuccessful` → body에서 `"id"` JSON 필드 추출 후 `Result.success(fileId ?: "")`
- `response.code == 401` → `Result.failure(UnauthorizedException("Drive access token 만료 — 재인증 필요"))`
- 그 외 → `Result.failure(IOException("Drive 업로드 실패: HTTP ${response.code}"))`
- Exception catch → `Result.failure(e)`

**JSON 파싱 (`parseFileId` private 메서드):** `org.json.JSONObject` (Android 플랫폼 API, 의존성 추가 불필요) 사용. `JSONObject(json).optString("id")`.

신규 의존성 추가 금지 — OkHttp 4.12.0, org.json 모두 이미 프로젝트에 있음.

**RepositoryModule.kt 수정** — 기존 abstract class에 아래 바인딩 추가. DriveUploadRepository는 인터페이스가 없는 `@Singleton` 클래스이므로 `@Provides` 방식으로 추가한다. RepositoryModule은 `abstract class`이므로 companion object에 `@Provides`를 배치한다:
```kotlin
companion object {
    @Provides
    @Singleton
    fun provideDriveUploadRepository(okHttpClient: OkHttpClient): DriveUploadRepository =
        DriveUploadRepository(okHttpClient)
}
```
필요한 import 추가:
- `import com.autominuting.data.drive.DriveUploadRepository`
- `import okhttp3.OkHttpClient`
- `import dagger.Provides`
- `import javax.inject.Singleton`

단, 기존 NetworkModule에 OkHttpClient @Provides가 있는지 먼저 확인한다. 있다면 companion object 패턴 대신 DriveUploadRepository에 `@Inject constructor`가 있으므로 RepositoryModule에 별도 바인딩이 불필요할 수 있다 — DriveUploadRepository 자체가 `@Singleton @Inject constructor`이면 Hilt가 자동으로 처리하므로 RepositoryModule 수정 없이 Task 1 구현만으로 충분하다. 컴파일 오류 여부를 확인 후 판단한다.
