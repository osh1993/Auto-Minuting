**PipelineNotificationHelper.kt 변경:**

`updateProgress` 메서드에 `progress: Int = -1` 파라미터를 추가한다. progress >= 0이면 determinate 프로그레스바, < 0이면 indeterminate:

```kotlin
fun updateProgress(context: Context, text: String, ongoing: Boolean = true, progress: Int = -1) {
    val builder = NotificationCompat.Builder(context, PIPELINE_CHANNEL_ID)
        .setSmallIcon(R.drawable.ic_launcher_foreground)
        .setContentTitle("Auto Minuting")
        .setContentText(text)
        .setOngoing(ongoing)
        .setSilent(true)

    if (progress >= 0) {
        builder.setProgress(100, progress, false)
    } else {
        builder.setProgress(0, 0, true)  // indeterminate
    }

    val notificationManager =
        context.getSystemService(Context.NOTIFICATION_SERVICE) as NotificationManager
    notificationManager.notify(PIPELINE_NOTIFICATION_ID, builder.build())
}
```

**DashboardViewModel.kt 변경:**

1. `transcriptionProgress: StateFlow<Float>` 프로퍼티를 추가한다. WorkManager의 WorkInfo를 관찰하여 진행률을 추출한다:

```kotlin
private val _transcriptionProgress = MutableStateFlow(0f)
val transcriptionProgress: StateFlow<Float> = _transcriptionProgress.asStateFlow()
```

2. TranscriptionTriggerWorker를 enqueue하는 모든 위치에서 WorkRequest의 id를 저장하고, WorkInfo를 관찰하는 로직을 추가한다:

```kotlin
private var currentTranscriptionWorkId: java.util.UUID? = null

// Worker enqueue 후:
currentTranscriptionWorkId = workRequest.id
observeTranscriptionProgress(workRequest.id)

private fun observeTranscriptionProgress(workId: java.util.UUID) {
    viewModelScope.launch {
        WorkManager.getInstance(context)
            .getWorkInfoByIdFlow(workId)
            .collect { workInfo ->
                if (workInfo != null) {
                    val progress = workInfo.progress.getInt(
                        TranscriptionTriggerWorker.KEY_PROGRESS, 0
                    )
                    _transcriptionProgress.value = progress / 100f

                    // 완료/실패 시 진행률 초기화
                    if (workInfo.state.isFinished) {
                        _transcriptionProgress.value = 0f
                        currentTranscriptionWorkId = null
                    }
                }
            }
    }
}
```

주의: DashboardViewModel에서 TranscriptionTriggerWorker를 enqueue하는 위치가 여러 곳일 수 있다 (URL 다운로드 등). 모든 enqueue 위치를 확인하고 observeTranscriptionProgress를 호출한다. ShareReceiverActivity에서 enqueue하는 경우는 이 ViewModel에서 관찰하지 않으므로 별도 처리 불필요 (알림에서 진행률 표시).
