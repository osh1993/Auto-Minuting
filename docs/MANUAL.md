## 1. 앱 소개

Auto Minuting은 Plaud 녹음기에서 녹음한 음성 파일을 받아 자동으로 회의록을 만들어주는 Android 앱입니다.

**전체 흐름:**

```
Plaud 녹음기 / 삼성 녹음앱 → (공유) → Auto Minuting → STT 전사 → AI → 회의록
```

**STT 엔진 5종 (설정에서 선택):**
- **Whisper 온디바이스**: 오프라인 동작, 무제한, 느림 (~500MB 모델 다운로드 필요)
- **Gemini Flash**: Google 클라우드, 한국어 지원, 빠름
- **Groq Whisper**: 빠른 클라우드 처리 (25MB 파일 제한)
- **Deepgram Nova-3**: 고정밀 인식
- **Naver CLOVA Speech**: 한국어 특화

**회의록 엔진 3종 (설정에서 선택):**
- **Gemini Flash**: 구조화된 Markdown 회의록
- **Deepgram Intelligence**: 텍스트 분석 기반 (영어 지원)
- **Naver CLOVA Summary**: 한국어 요약

## 2. 설치 방법 (비스토어 배포)

앱이 Google Play Store에 등록되어 있지 않은 경우, APK 파일을 직접 설치합니다. 이를 **사이드로딩(Sideloading)** 이라 합니다.

### 2.1 APK 파일 다운로드

[GitHub Releases](https://github.com/osh1993/Auto-Minuting/releases)에서 최신 APK를 다운로드합니다.

| 버전 | 주요 기능 |
| -- | -- |
| v9.0 | 다중 Gemini API 키, 홈 화면 로컬 파일 직접 입력, Groq 대용량 자동 청크 분할, MP3 다중 합치기 |
| v8.0 | 다중 M4A 파일 자동 합치기, Groq 크기 초과 엔진 변경 안내 |
| v7.0 | Google Drive 연동, 회의록 편집, API 사용량 대시보드 |
| v6.0 | 멀티 STT/회의록 엔진, API 키 보안 강화 |

### 2.2 APK 파일 전달 방법

**방법 A — Google Drive / OneDrive로 전달**
1. APK 파일을 클라우드 드라이브에 업로드
2. 기기에서 드라이브 앱을 열어 APK 파일 다운로드
3. 다운로드 완료 후 탭하여 설치 시작

**방법 B — USB 케이블로 직접 전달**
1. USB 케이블로 PC와 기기 연결
2. 기기에서 "파일 전송(MTP)" 모드 선택
3. APK 파일을 기기의 Download 폴더에 복사
4. 기기 파일 관리자 앱에서 APK 파일 탭하여 설치

**방법 C — ADB로 직접 설치 (개발자용)**
```bash
adb install AutoMinuting-v9.0-release.apk
```

### 2.3 "출처를 알 수 없는 앱" 허용

**Android 12 이상 (삼성 갤럭시 기준):**
1. `설정` → `생체 인식 및 보안` → `출처를 알 수 없는 앱 설치`
2. APK를 다운로드한 앱(예: Chrome, 파일 관리자) 선택
3. "이 출처의 앱 허용" 토글 ON

> **보안 주의**: 신뢰할 수 있는 출처의 APK만 설치하세요. 설치 완료 후 해당 토글을 다시 OFF로 되돌리는 것을 권장합니다.

### 2.4 설치 진행

1. APK 파일을 탭하면 설치 화면이 나타납니다
2. "설치" 버튼을 탭합니다
3. "Play Protect 경고"가 나타나면 "설치" 탭 (개인 개발 앱이라 Play 검증을 통과하지 않아 표시됨)
4. 설치 완료 후 앱 목록에서 "Auto Minuting" 확인

> **이전 버전(debug 빌드)이 설치된 경우**: release 빌드와 서명이 달라 덮어쓰기 설치가 안 됩니다. 이전 버전을 먼저 삭제 후 설치하세요.

## 3. 초기 설정

앱을 처음 실행하면 정상 동작을 위해 아래 설정이 필요합니다.

### 3.1 STT 엔진 선택 및 API 키 설정

`설정` → "STT 엔진" 에서 사용할 전사 엔진을 선택합니다.

| 엔진 | API 키 필요 | 파일 크기 제한 | 특징 |
|------|------------|----------------|------|
| Whisper 온디바이스 | 없음 | 없음 | 오프라인, 느림 |
| Gemini Flash | Google OAuth 또는 API 키 | 없음 | 빠름, 한국어 우수 |
| Groq Whisper | Groq API 키 | **25MB** | 매우 빠름 |
| Deepgram Nova-3 | Deepgram API 키 | 없음 | 고정밀 |
| Naver CLOVA | Naver API 키 | 없음 | 한국어 특화 |

#### Gemini Flash 설정

**방법 A — Google 계정 (OAuth):**
1. `설정` → "Google 계정" 섹션에서 인증 방식 "Google 계정 (OAuth)" 선택
2. "Google OAuth Web Client ID" 입력 필드에 Web Client ID 입력 후 저장
   - Google Cloud Console → APIs & Services → Credentials에서 **웹 애플리케이션 타입** OAuth 클라이언트의 Client ID 복사
3. "Google 계정으로 로그인" 버튼 탭

**방법 B — API 키:**
1. `설정` → "Google 계정" 섹션에서 인증 방식 "API 키" 선택
2. [Google AI Studio](https://aistudio.google.com/)에서 API 키 발급
3. "Gemini API 키" 항목에 붙여넣기 후 저장

#### Groq Whisper 설정

1. [console.groq.com](https://console.groq.com)에서 API 키 발급
2. `설정` → "Groq API 키" 항목에 입력 후 저장
3. ⚠️ **파일 크기 25MB 제한** — 여러 파일을 합친 경우 초과 시 다른 엔진으로 변경 필요

#### Deepgram 설정

1. [deepgram.com](https://deepgram.com)에서 API 키 발급
2. `설정` → "Deepgram API 키" 항목에 입력 후 저장

#### Naver CLOVA 설정

1. [NAVER Cloud Platform](https://www.ncloud.com)에서 CLOVA Speech API 키 발급
2. `설정` → "Naver API 키" 항목에 입력 후 저장

#### Whisper 온디바이스 설정

1. `설정` → "Whisper 모델 다운로드" 탭
2. 약 500MB 다운로드 (Wi-Fi 권장)
3. 다운로드 완료 후 온디바이스 전사 사용 가능

### 3.2 회의록 엔진 선택

`설정` → "회의록 엔진" 에서 선택합니다.

| 엔진 | 특징 |
|------|------|
| Gemini Flash | 구조화된 Markdown 회의록, 한국어 |
| Deepgram Intelligence | 텍스트 분석 기반 (영어 지원) |
| Naver CLOVA Summary | 한국어 요약 |

### 3.3 알림 권한 허용 (Android 13 이상)

앱이 처음 실행될 때 알림 권한을 요청합니다. "허용"을 탭하면 전사/회의록 완료 알림을 받을 수 있습니다.

> **중요**: 알림 권한이 꺼져 있으면 전사 완료/오류 알림을 받을 수 없습니다. `설정 → 앱 → Auto Minuting → 알림 → 허용`에서 활성화하세요.

### 3.4 Google Drive 연동 (선택)

회의록을 Google Drive에 자동 업로드하려면:

1. `설정` → "Google Drive" 섹션에서 "Google Drive 연결" 탭
2. Google 계정 선택 → Drive 접근 동의
3. (선택) "Drive 폴더 ID" 입력: 특정 폴더에 저장하려면 폴더 URL의 ID 부분 입력
4. "Drive 자동 업로드" 토글 ON

## 4. 주요 기능

### 4.1 음성 입력 방법

#### 방법 1 — URL로 음성 다운로드

1. `대시보드` 탭 상단의 URL 입력 필드에 음성 파일 URL 붙여넣기
2. 다운로드 버튼 탭
3. 다운로드 완료 후 자동으로 전사 시작

#### 방법 2 — Plaud 공유 링크 수신

1. 다른 앱(카카오톡, 메일 등)에서 web.plaud.ai 링크를 공유
2. 공유 대상 목록에서 "Auto Minuting" 선택
3. 오디오 자동 추출 후 전사 파이프라인 시작

#### 방법 3 — 삼성 녹음앱에서 단일 파일 공유

1. 삼성 녹음앱에서 녹음 파일 하나 선택
2. 공유 버튼 → "Auto Minuting" 선택
3. 자동으로 전사 파이프라인 시작

#### 방법 4 — 삼성 녹음앱에서 여러 파일 동시 공유 (v8.0+)

1. 삼성 녹음앱에서 여러 녹음 파일을 다중 선택
2. 공유 버튼 → "Auto Minuting" 선택
3. 파일들이 자동으로 하나의 파일로 합쳐진 후 전사 파이프라인 시작 (M4A: MediaMuxer, MP3: 재인코딩 없음, 혼재: 포맷별 분리)
4. 합쳐진 파일의 이름은 **첫 번째 파일명** 사용

> **참고**: 파일 순서는 선택 순서를 따릅니다.

#### 방법 5 — 홈 화면에서 로컬 파일 직접 선택 (v9.0)

1. `대시보드` 탭 → "URL에서 음성 다운로드" 카드 아래쪽 "로컬 파일 불러오기" 섹션
2. **파일 불러오기** 버튼 탭 → 파일 피커에서 M4A/MP3 선택
3. 회의 제목 입력 후 **확인** 탭 (기본값: 파일명)
4. 기존 파이프라인(STT → 회의록)이 자동 시작됨

### 4.2 전사 텍스트 직접 입력

이미 전사된 텍스트가 있는 경우, STT 과정 없이 바로 회의록 생성 파이프라인에 진입할 수 있습니다.

**방법 A — 텍스트 파일 공유**
1. 파일 관리자에서 `.txt` 파일 선택
2. 공유 → "Auto Minuting" 선택
3. 텍스트가 전사본으로 저장되어 회의록 생성 준비 완료

**방법 B — 텍스트 직접 공유 (EXTRA_TEXT)**
1. 메모 앱, 브라우저 등에서 텍스트를 선택하여 공유
2. "Auto Minuting" 선택
3. 전사본으로 저장 후 회의록 생성 준비 완료

**방법 C — 여러 텍스트 파일 동시 공유**
1. 파일 관리자에서 여러 텍스트 파일을 다중 선택하여 공유
2. "Auto Minuting" 선택
3. 파일 내용이 하나로 합쳐져 전사본으로 저장됨

### 4.3 전사 목록 (Transcripts 탭)

**카드 정보:**
- 파일 이름 (탭하여 편집 가능)
- 파일 종류 아이콘 (텍스트 / 음성)
- 전사 상태 배지 (전사 완료 / 전사 중 / 실패)
- 연결된 회의록 수 배지 (1개 이상일 때 표시)
- 생성 날짜 (yyyy.MM.dd HH:mm)

**카드 액션 메뉴 (⋮ 탭):**
- **회의록 작성**: 전사 완료 파일에서 회의록 생성
- **추가 생성**: 이미 회의록이 있을 때 새 회의록을 추가로 생성 (기존 회의록 보존)
- **재전사**: 전사를 다시 실행
- **공유**: 전사 텍스트를 다른 앱으로 공유
- **삭제**: 전사 파일 삭제 (연결된 회의록은 보존됨)

### 4.4 회의록 목록 (Minutes 탭)

**카드 정보:**
- 회의록 제목 (AI가 자동 추출, 탭하여 편집 가능)
- 출처 전사 이름 (탭하면 해당 전사로 이동)
- 생성 날짜 (yyyy.MM.dd HH:mm)

**카드 액션 메뉴 (⋮ 탭):**
- **편집**: 회의록 내용 직접 편집
- **공유**: 회의록을 Markdown 형식으로 다른 앱에 공유
- **삭제**: 회의록만 삭제 (전사 파일은 보존됨)

**회의록 내용 보기**: 카드를 탭하면 Markdown 형식으로 렌더링된 회의록 전문을 확인할 수 있습니다.

### 4.5 회의록 생성 시 프롬프트 선택

회의록 작성 버튼을 탭하면 프롬프트 템플릿 선택 다이얼로그가 표시됩니다.

**기본 제공 템플릿:**
- **구조화된 회의록**: 안건, 논의 내용, 결정 사항, 액션 아이템 형식
- **요약 회의록**: 핵심 내용만 간략하게 요약
- **액션 아이템**: 담당자별 할 일 목록 중심

**기본 템플릿 고정**: `설정` → "기본 프롬프트"에서 특정 템플릿을 선택하면 다이얼로그 없이 자동 생성됩니다.

### 4.6 API 사용량 대시보드

`대시보드` 탭에서 각 엔진별 API 호출 횟수를 확인할 수 있습니다.

- STT 엔진별 / 회의록 엔진별 누적 사용 횟수
- Gemini Free 티어 쿼터 사용량 및 90% 초과 시 경고 배너

## 5. 파이프라인 상세

### 5.1 완전 자동 모드

`설정` → "자동 회의록 생성" ON 상태.

```
음성 입력 → 전사 완료 → 회의록 자동 생성 (기본 템플릿 적용)
```

모든 단계가 자동으로 진행되며, 각 단계 완료 시 알림이 표시됩니다.

### 5.2 하이브리드 모드

`설정` → "자동 회의록 생성" OFF 상태.

```
음성 입력 → 전사 완료 → 대시보드에 "회의록 생성" 버튼 표시 → 사용자 확인 → 회의록 생성
```

전사는 자동으로 이루어지지만, 회의록 생성은 사용자가 확인 버튼을 탭해야 시작됩니다.

### 5.3 다중 파일 합치기 (v8.0+)

여러 파일을 동시에 공유하면:

```
여러 파일 공유 → 자동 합치기 → 단일 파일 → STT → 회의록
```

- M4A: MediaMuxer (재인코딩 없음)
- MP3: 재인코딩 없이 합치기 (v9.0)
- M4A+MP3 혼재: 포맷별 분리 처리 (v9.0)
- 파일 순서: 공유 시 선택한 순서 그대로
- 합쳐진 파일명: 첫 번째 파일명 사용

### 5.4 Groq 대용량 파일 자동 분할 전사 (v9.0)

Groq Whisper 엔진 사용 시 25MB 초과 파일도 자동 처리됩니다.

```text
파일 수신 → 25MB 초과 감지 → 600초 단위 청크 자동 분할 → 순차 전사 → 결과 이어붙이기
```

- 분할/이어붙이기가 백그라운드에서 자동 진행 (별도 조작 불필요)
- 기존 파이프라인과 동일한 결과물 생성

### 5.5 전사-회의록 독립 구조

v5.0부터 전사 파일과 회의록 파일은 완전히 독립적으로 관리됩니다.

- 전사 파일 삭제 → 연결된 회의록은 보존
- 회의록 삭제 → 전사 파일 및 상태 무영향

## 7. 문제 해결

### Google 계정 로그인 실패: "No credentials available"

Release APK의 서명 키(SHA-1)가 Google Cloud Console에 미등록된 경우 발생합니다.

**해결:**
1. [Google Cloud Console](https://console.cloud.google.com) → APIs & Services → Credentials
2. Android 타입 OAuth 클라이언트에 Release APK SHA-1 추가
   - Package name: `com.autominuting`
   - SHA-1: `22:1C:FD:DB:E8:CE:66:B7:A0:36:23:FB:73:D8:EF:B2:2F:03:A6:76`

> **참고**: Debug APK와 Release APK의 SHA-1이 다릅니다. 두 환경을 모두 사용한다면 각각 등록해야 합니다.

### Google 계정 로그인 실패: "[28444] Developer console is not set up correctly"

Web Client ID가 없거나 잘못된 값(예: Android 타입 ID를 입력한 경우)입니다.

**해결:**
1. [Google Cloud Console](https://console.cloud.google.com) → APIs & Services → Credentials
2. OAuth 2.0 클라이언트 목록에서 **"웹 애플리케이션" 타입** 클라이언트 확인
   - 없으면: "Create Credentials" → "OAuth client ID" → Application type: **Web application** 으로 생성
3. 웹 애플리케이션 타입 Client ID 복사 (`숫자-영숫자.apps.googleusercontent.com` 형태)
4. 앱 `설정` → "Google OAuth Web Client ID" 입력 필드에 붙여넣기 → "저장"
5. "Google 계정으로 로그인" 버튼 재시도

> ⚠️ **Android 타입 Client ID와 혼동 주의**: `setServerClientId()`에는 반드시 **웹 애플리케이션** 타입 ID를 입력해야 합니다.

### Groq 전사 실패: 파일 크기 초과

v9.0부터 Groq Whisper는 **25MB 초과 파일을 자동으로 청크 분할**하여 처리합니다. 별도 조작 없이 자동 진행됩니다.

분할 전사 중에도 문제가 발생하면:

- `설정` → STT 엔진을 **Whisper 온디바이스** 또는 **Gemini Flash**로 변경

### 전사가 오래 걸리거나 완료되지 않는 경우

- **Whisper 사용 시**: 긴 녹음 파일은 수십 분이 소요될 수 있습니다. 알림 표시줄에서 진행률을 확인하세요.
- **클라우드 STT 사용 시**: 인터넷 연결을 확인하세요.
- **앱 종료 후 재시작**: 비정상 종료 시 앱을 다시 열면 중단된 전사 상태가 "실패"로 전환됩니다. 카드 액션 메뉴에서 "재전사"를 선택하세요.

### 회의록 생성이 실패하는 경우

- 선택한 회의록 엔진의 API 키가 올바른지 확인하세요
- Gemini Free 티어 일일 한도 초과 시 다음 날 자동 초기화됩니다
- 대시보드의 API 사용량 위젯에서 쿼터 상태를 확인하세요

### 알림이 표시되지 않는 경우

`설정 → 앱 → Auto Minuting → 알림 → 허용`에서 알림 권한을 활성화하세요.

### APK 설치 시 "기존 버전과 충돌" 오류

Debug 빌드와 Release 빌드는 서명이 달라 덮어쓰기 설치가 불가합니다.

**해결:** 기존 앱을 완전히 삭제 후 재설치하세요. (기존 데이터 삭제됨에 주의)

### APK 설치 후 앱이 실행되지 않는 경우

Android 12 미만 기기는 지원하지 않습니다 (minSdk 31).
