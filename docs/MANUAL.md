## 1. 앱 소개

Auto Minuting은 Plaud 녹음기에서 녹음한 음성 파일을 받아 자동으로 회의록을 만들어주는 Android 앱입니다.

**전체 흐름:**

```
Plaud 녹음기 → (BLE/공유) → Auto Minuting → STT 전사 → Gemini AI → 회의록
```

**STT 엔진 두 가지:**
- **Gemini 클라우드 STT**: 인터넷 연결 필요, 빠름
- **Whisper 온디바이스 STT**: 오프라인 동작, 느리지만 인터넷 불필요

## 2. 설치 방법 (비스토어 배포)

앱이 Google Play Store에 등록되어 있지 않은 경우, APK 파일을 직접 설치합니다. 이를 **사이드로딩(Sideloading)** 이라 합니다.

### 2.1 APK 파일 전달 방법

다음 중 하나로 APK 파일을 기기에 전달합니다.

**방법 A — Google Drive / OneDrive로 전달**
1. APK 파일을 클라우드 드라이브에 업로드
2. 기기에서 드라이브 앱을 열어 APK 파일 다운로드
3. 다운로드 완료 후 탭하여 설치 시작

**방법 B — USB 케이블로 직접 전달**
1. USB 케이블로 PC와 기기 연결
2. 기기에서 "파일 전송(MTP)" 모드 선택
3. APK 파일을 기기의 Download 폴더에 복사
4. 기기 파일 관리자 앱에서 APK 파일 탭하여 설치

**방법 C — ADB로 직접 설치 (개발자용)**
```bash
# USB 연결 + USB 디버깅 활성화 후
adb install app-release.apk
```

**방법 D — QR코드 / 링크 배포**
1. APK 파일을 웹 서버 또는 파일 공유 서비스에 업로드
2. 다운로드 링크를 QR코드로 변환 (qr-code-generator.com 등)
3. 수신자가 QR코드를 스캔하여 기기에서 직접 다운로드 후 설치

### 2.2 "출처를 알 수 없는 앱" 허용

APK를 처음 설치할 때 Android 보안 설정에서 허용해야 합니다.

**Android 12 이상 (삼성 갤럭시 기준):**
1. `설정` → `생체 인식 및 보안` → `출처를 알 수 없는 앱 설치`
2. APK를 다운로드한 앱(예: Chrome, 파일 관리자) 선택
3. "이 출처의 앱 허용" 토글 ON

> **보안 주의**: 신뢰할 수 있는 출처의 APK만 설치하세요. 설치 완료 후 해당 토글을 다시 OFF로 되돌리는 것을 권장합니다.

### 2.3 설치 진행

1. APK 파일을 탭하면 설치 화면이 나타납니다
2. "설치" 버튼을 탭합니다
3. "Play Protect 경고"가 나타나면 "설치" 탭 (개인 개발 앱이라 서명이 Play 검증을 통과하지 않아 표시됨)
4. 설치 완료 후 앱 목록에서 "Auto Minuting" 확인

## 3. 초기 설정

앱을 처음 실행하면 정상 동작을 위해 아래 설정이 필요합니다.

### 3.1 Gemini API 키 설정 (필수)

Gemini AI로 회의록을 생성하려면 API 키가 필요합니다.

**API 키 발급:**
1. [Google AI Studio](https://aistudio.google.com/)에 접속
2. "Get API key" → "Create API key" 클릭
3. 생성된 키 복사 (AIza로 시작하는 문자열)

**앱에 입력:**
1. 앱 하단 탭에서 `설정` 탭 이동
2. "Gemini API 키" 항목 탭
3. 복사한 키 붙여넣기 후 저장

### 3.2 알림 권한 허용 (Android 13 이상)

앱이 처음 실행될 때 알림 권한을 요청합니다. "허용"을 탭하면 전사/회의록 완료 알림을 받을 수 있습니다.

### 3.3 STT 엔진 선택

`설정` → "STT 엔진" 에서 선택합니다.

| 엔진 | 장점 | 단점 |
|------|------|------|
| Gemini 클라우드 | 빠름, 정확도 높음 | 인터넷 필요, API 호출 소모 |
| Whisper 온디바이스 | 오프라인 동작, 무제한 | 느림 (파일 길이에 비례), 모델 다운로드 필요 |

**Whisper 사용 시 모델 다운로드:**
- `설정` → "Whisper 모델 다운로드" 탭
- 약 500MB 다운로드 (Wi-Fi 권장)
- 다운로드 완료 후 온디바이스 전사 사용 가능

## 4. 주요 기능

### 4.1 음성 입력 방법

**방법 1 — Plaud BLE 연결**
- 앱 `대시보드` 탭에서 Plaud 녹음기와 BLE 페어링
- 연결 후 녹음기의 파일 목록을 불러와 전사 파이프라인에 자동 진입

**방법 2 — URL로 음성 다운로드**
1. `대시보드` 탭 상단의 URL 입력 필드에 음성 파일 URL 붙여넣기
2. 다운로드 버튼 탭
3. 다운로드 완료 후 자동으로 전사 시작

**방법 3 — Plaud 공유 링크 수신**
1. 다른 앱(카카오톡, 메일 등)에서 web.plaud.ai 링크를 공유
2. 공유 대상 목록에서 "Auto Minuting" 선택
3. 오디오 자동 추출 후 전사 파이프라인 시작

**방법 4 — 삼성 녹음앱에서 공유**
1. 삼성 녹음앱에서 녹음 파일 선택
2. 공유 버튼 → "Auto Minuting" 선택
3. 자동으로 전사 파이프라인 시작

### 4.2 전사 목록 (Transcripts 탭)

전사된 모든 파일 목록을 확인합니다.

**카드 정보:**
- 파일 이름 (탭하여 편집 가능)
- 파일 종류 아이콘 (텍스트 / 음성)
- 전사 상태 배지 (전사 완료 / 전사 중 / 실패)
- 연결된 회의록 수 배지 (1개 이상일 때 표시)
- 생성 날짜 (yyyy.MM.dd HH:mm)

**카드 액션 메뉴 (⋮ 탭):**
- **회의록 작성**: 전사 완료 파일에서 회의록 생성
- **추가 생성**: 이미 회의록이 있을 때 새 회의록을 추가로 생성 (기존 회의록 보존)
- **재전사**: 전사를 다시 실행
- **공유**: 전사 텍스트를 다른 앱으로 공유
- **삭제**: 전사 파일 삭제 (연결된 회의록은 보존됨)

### 4.3 회의록 목록 (Minutes 탭)

생성된 모든 회의록 목록을 확인합니다.

**카드 정보:**
- 회의록 제목 (Gemini가 자동 추출, 탭하여 편집 가능)
- 출처 전사 이름 (탭하면 해당 전사로 이동)
- 생성 날짜 (yyyy.MM.dd HH:mm)

**카드 액션 메뉴 (⋮ 탭):**
- **공유**: 회의록을 Markdown 형식으로 다른 앱에 공유
- **NotebookLM에서 열기**: NotebookLM 앱 또는 웹으로 이동
- **삭제**: 회의록만 삭제 (전사 파일은 보존됨)

**회의록 내용 보기**: 카드를 탭하면 Markdown 형식으로 렌더링된 회의록 전문을 확인할 수 있습니다.

### 4.4 회의록 생성 시 프롬프트 선택

회의록 작성 버튼을 탭하면 프롬프트 템플릿 선택 다이얼로그가 표시됩니다.

**기본 제공 템플릿:**
- **구조화된 회의록**: 안건, 논의 내용, 결정 사항, 액션 아이템 형식
- **요약 회의록**: 핵심 내용만 간략하게 요약
- **액션 아이템**: 담당자별 할 일 목록 중심

**기본 템플릿 고정 설정**: `설정` → "기본 프롬프트" 에서 특정 템플릿을 선택하면 다이얼로그 없이 해당 템플릿으로 자동 생성됩니다.

### 4.5 Gemini 쿼터 모니터링

`대시보드` 탭에서 Gemini Free 티어의 API 호출 사용량을 확인할 수 있습니다.

- STT 호출 / 회의록 생성 호출 구분 표시
- 사용량 90% 초과 시 경고 배너 표시
- Free 티어 한도 초과 시 자동 재시도 (일시적 한도 초과인 경우)

## 5. 파이프라인 상세

### 5.1 완전 자동 모드

`설정` → "자동 회의록 생성" ON 상태.

```
음성 입력 → 전사 완료 → 회의록 자동 생성 (기본 템플릿 적용)
```

모든 단계가 자동으로 진행되며, 각 단계 완료 시 알림이 표시됩니다.

### 5.2 하이브리드 모드

`설정` → "자동 회의록 생성" OFF 상태.

```
음성 입력 → 전사 완료 → 대시보드에 "회의록 생성" 버튼 표시 → 사용자 확인 → 회의록 생성
```

전사는 자동으로 이루어지지만, 회의록 생성은 사용자가 확인 버튼을 탭해야 시작됩니다.

### 5.3 전사-회의록 독립 구조

v5.0부터 전사 파일과 회의록 파일은 완전히 독립적으로 관리됩니다.

- 전사 파일 삭제 → 연결된 회의록은 보존
- 회의록 삭제 → 전사 파일 및 상태 무영향
- 같은 전사에서 회의록 재생성 → 기존 회의록 보존 + 새 회의록 추가

## 6. 설정 화면 안내

| 항목 | 설명 |
|------|------|
| Gemini API 키 | Gemini API 인증 키 (암호화 저장) |

| 커스텀 프롬프트 | 직접 프롬프트를 입력하여 기본으로 설정 |

## 7. 문제 해결

### 전사가 오래 걸리거나 완료되지 않는 경우

- **Whisper 사용 시**: 긴 녹음 파일은 수십 분이 소요될 수 있습니다. 알림 표시줄에서 진행률을 확인하세요.
- **Gemini STT 사용 시**: 인터넷 연결을 확인하세요.
- **앱 종료 후 재시작**: 비정상 종료 시 앱을 다시 열면 중단된 전사 상태가 "실패"로 전환됩니다. 카드 액션 메뉴에서 "재전사"를 선택하세요.

### 회의록 생성이 실패하는 경우

- Gemini API 키가 올바른지 확인하세요 (`설정` → "Gemini API 키")
- API 쿼터를 확인하세요 (대시보드의 쿼터 위젯)
- Free 티어 일일 한도 초과 시 다음 날 자동으로 초기화됩니다

### APK 설치 후 앱이 실행되지 않는 경우

- Android 12 미만 기기는 지원하지 않습니다 (minSdk 31)
- "이 기기에 설치할 수 없음" 메시지 시: 기기의 Android 버전을 확인하세요
